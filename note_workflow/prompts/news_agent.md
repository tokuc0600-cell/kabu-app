# news_agent.md — 時事ネタ記事生成プロンプト（Leon版）

## このプロンプトの用途

スプレッドシートに収集した海外IR・企業ニュース・マーケット情報をもとに、
投資初心者にも読みやすい「時事ネタ解説記事」を生成するためのプロンプト。

**note_agent.md（開発トレース記事）との使い分け：**
- 開発体験・kabu_appの話 → `note_agent.md`
- 企業ニュース・市況・IR情報の解説 → **このファイル（news_agent.md）**

---

## 依頼時に必ず指定する情報

Claude Codeへの依頼時に以下を伝えること。

```
- 処理対象のファイルパス（drafts/news-[slug].md）
- 記事のスラッグ（例：news-accenture-q3-earnings）
- ニュースの種類（M&A / リストラ / 事業譲渡 / 決算 / その他）
- 対象企業・業界（例：アクセンチュア、ITコンサル業界）
```

**フローの補足：**
- ドラフトはClaude.aiが生成し `.md` ファイルとして出力する
- レオンさんがそのファイルを `drafts/` に手動配置する
- Claude Codeは配置済みの `drafts/[slug].md` を読み込んで仕上げる
- Google DocsファイルIDの指定は不要（旧フローの名残）

## 記事生成の指示

### ゴール

投資に興味はあるけれど難しそうと感じている初心者が、「これなら読める」「自分にも関係あるかも」と感じられる記事にする。専門用語は使う場合は必ず平易な言葉で補足する。

### 記事の構成（標準テンプレート）

```
## [記事タイトル]
（タイトルは「○○が××した理由」「△△から学ぶ投資の教訓」など、
  具体的な企業名＋読者へのメリットが伝わる形にする）

（画像挿入：記事のテーマを象徴するアイキャッチ）

### はじめに
- 今回取り上げるニュースの概要を2〜3文で
- 「なぜ個人投資家として注目したのか」をレオンの言葉で添える

### [ニュースの種類に応じたメインセクション]
※ 種類別のセクション構成は後述

### 投資家として見るべきポイント
- ポジションなしの場合：どう様子を見るか
- ポジションありの場合：リスクヘッジの考え方
（必ず両立場を書き分けること）

（画像挿入：投資判断のフローや注目指標の図解）

### まとめ
- 記事全体の要点を箇条書き（3〜5点）
- 締めはレオンの一言コメントで終わる（決意表明や感想など、人間らしい言葉で）
```

## ニュースの種類別・メインセクションの構成

### M&A（買収・合併）の場合

```
### [企業名]が[ターゲット]を買収した背景
- 買収の概要（金額・スキーム・タイムライン）
  （画像挿入：買収スキームの図解）
- なぜ今この買収なのか（業界トレンド・競合との関係）

### 買収後に何が変わるのか
- 買収企業・被買収企業それぞれへの影響
- 顧客・サービスへの影響

### 会計・財務への影響
- のれん計上の可能性
- 短期的なPLへの影響（一時費用など）
  （画像挿入：財務影響の比較表 PNG画像化）
```

### リストラ・人員削減の場合

```
### 何が起きたのか
- 削減規模・対象部門・タイムライン
- 発表の背景（業績悪化・戦略転換・コスト最適化）

### なぜリストラに踏み切ったのか
- 経営判断の背景を読む
- 同業他社の動向との比較

### 決算数値への短期・中長期の影響
- 一時費用（リストラ費用）の計上タイミング
- 固定費削減による営業利益率改善の見通し
  （画像挿入：リストラ前後の収益構造の変化イメージ図）
```

### 事業譲渡・売却の場合

```
### 何を売ったのか・なぜ売ったのか
- 売却事業の概要・売却額
- 「選択と集中」の文脈での位置づけ

### 残る事業の収益力はどう変わるか
- 売却後のセグメント構成の変化
- 継続事業の利益率・成長性の見通し
  （画像挿入：売却前後の事業ポートフォリオ比較 PNG画像化）

### 会計上の表示に注意（基準別の違い）
- 日本基準・IFRS・米国基準で表示がどう変わるか（該当する場合のみ）
```

### 決算・業績発表の場合

```
### 数字のサマリー
- 売上・営業利益・純利益の前年比
- コンセンサス予想との比較
  （画像挿入：業績推移グラフ PNG画像化）

### 数字の"裏側"を読む
- 一時費用・特別損益があれば除いた実力値はどうか
- セグメント別で伸びている／落ちているのはどこか

### 次の四半期・通期見通しのポイント
- 会社ガイダンスの読み方
- アナリストが注目しているKPI
```

## 執筆上の注意事項

### 必ず守ること

- 文末は**です・ます調**で統一する
- 一人称は「僕」、二人称は使わない
- 話し言葉に近い、平易な日本語を使う
- 専門用語（のれん・EV/EBITDA・フリーキャッシュフローなど）は初出時に必ずカッコ書きで補足する
- 市況・投資判断については断定を避け、「〜かもしれません」「〜な気がします」などの控えめな表現を使う
- リストラ・人員削減を扱う場合、「組織スリム化」「効率化」などの企業側の軽い言い回しを避け、影響を受ける人の立場が伝わる表現を選ぶ
- 投資アドバイスは「ポジションなし」「ポジションあり」の2パターンを必ず書き分ける

### 禁止事項

- 「〜すべきです」「〜に違いありません」などの断定的な投資アドバイスは書かない
- 特定銘柄の売買を強く推奨する表現は使わない
- note本文中にMarkdownの表を直接埋め込まない（必ずPNG化プレースホルダーに置き換える）

## 記事の長さの目安

| ニュースの種類 | 目安文字数 |
|---|---|
| M&A | 2,000〜3,000文字 |
| リストラ・人員削減 | 1,500〜2,500文字 |
| 事業譲渡・売却 | 1,500〜2,500文字 |
| 決算 | 1,500〜2,000文字 |

## 画像挿入の判断基準

以下のような箇所には積極的にプレースホルダーを挿入する。

- セクションの切り替わり（読者の視線を区切る）
- 比較・対照が必要な情報（表形式の内容 → 必ずPNG化）
- フローや因果関係（スキーム図・因果図）
- 数値データ（グラフ化した方が伝わる場合）
- まとめパート（記事を締めくくる図解）

## Geminiの案の扱い方

Geminiが生成した記事案はあくまでたたき台。以下の優先順位で仕上げる。

1. **ファクトの正確性を確認**：数値・企業名・日付に誤りがないかチェック
2. **構成をこのテンプレートに合わせて組み直す**：Geminiの構成をそのまま使わなくてよい
3. **レオンのペルソナに合わせて全文書き直す**：文体・一人称・視点をすべてレオンのものに統一する
4. **投資初心者目線で読み直す**：「これ、初めて読む人に伝わるか？」を自問しながら推敲する
