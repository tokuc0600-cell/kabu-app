# dev_logs について

開発トレース記事（`drafts/pf-backtest-engine.md` 等）に書き起こす前の、セッションごとの中間素材を置く場所。

## 運用ルール

1. kabu_appの開発が一区切りついたタイミング（動いた・検証できた・トラブルが解決した、等）で、
   Claude Codeに「今回のセッションのメモをdev_logsに残して」と伝える。
2. `YYYY-MM-DD-トピックslug.md` というファイル名で、以下のテンプレートに沿って書き出す。
3. 記事を書く/追記するタイミングで、該当メモを読み込んで記事の節として書き起こす。
4. 反映したらメモの末尾に `→ [記事slug]の「節タイトル」に反映済み` を追記する。

## テンプレート

```markdown
# YYYY-MM-DD トピック概要

## 概要
（1〜2行）

## 実際に出した指示
（生の文面をそのまま。整形しない）

## 起きたこと・気づいたこと

## 原因

## 直し方

## トラブルの詳細（あれば）
```

メモ自体は「です・ます調」などnote記事の執筆ルールに従わなくてよい（読み物ではなく素材のため）。
IDEの画面をそのままコピー&ペーストするのではなく、その場でClaude Codeに整理させて書き出すこと
（Markdownのエスケープが崩れず、後で記事に書き起こしやすい形になる）。
