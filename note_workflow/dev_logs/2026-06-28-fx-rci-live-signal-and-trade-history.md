# 2026-06-28 FXライブシグナルをRCI戦略に切り替え・トレード履歴機能を追加

## 概要

FXダッシュボードのライブシグナル判定をEMAクロスからRCI（短期反転）に切り替え、
エントリー/エグジット発生時にGoogle Sheetsへ自動記録＋Resendでメール通知する仕組みを実装した。
合わせてStreamlitにトレード履歴タブ・手動クローズボタンを追加し、
スマホ含む任意の端末からweb完結で運用できる体制を整えた。

## 起きたこと・気づいたこと

- セッション冒頭、2760（東京エレクトロン デバイス）のエグジット通知メールが2通届いた理由を調査した。
  `check_exit_signals.yml`に14:00 JST（intraday）と15:30 JST（close）の2スケジュールが設定されており、
  重複通知防止の仕組みが意図的に省かれているためだった。
  さらに「自分では何もしていないのにロング中になっていた」理由は`sync_kabu.py`の自動エントリー記録、
  「損切り条件に達しているのにクローズされない」理由はSheetsのH列（損切り%）が未入力のため
  `step_position()`の`if stop_loss_pct:`ガードがfalsyで判定をスキップしていたことが原因だった。

- EMAクロスとRCIの戦略比較を整理する過程で、EMAはトレンドフォロー型（少ない取引・長期ホールド）、
  RCIは波動の往復型（多い取引・短期ホールド）という性格の違いが明確になった。
  強いトレンド相場ではEMAが有利、レンジ相場ではRCIが有利という判断基準も整理できた。

- FXの自動同期（`sync_fx.yml`）を止めてRCI手動運用に切り替えるにあたり、
  「EMAロジックで動く既存の『🔄更新するボタン』がRCIで手動エントリーしたポジションを
  EMAデッドクロスで上書きクローズしてしまう」という衝突問題を事前に特定できた。
  これを解消するために同ボタン自体もRCIロジックに変更し、シグナル時間足セレクタを追加した。

- シグナル検出用時間足を1h/4hに限定する際、既存の`FX_INTERVAL_OPTIONS`（チャート表示用、1m〜週足）は
  変更せず、`SIGNAL_INTERVAL_OPTIONS`という別定数を新設した。チャートの短期足表示は
  エントリー直前の値動き確認に引き続き有用なため、削除しないことを選択。

- `sync_fx.py`でのRCI信号生成は`detect_rci_signal_series()`の戻り値最終行を取るだけでよく、
  既存の`step_position()`呼び出し（エントリー/エグジット/損切り/利確を一括管理）をそのまま流用できた。
  アーキテクチャ上の変更は「EMAクロス→RCIシグナル」の差し替え1箇所のみ。

- `app_fx.py`のポジション操作セクションには既にエントリーボタン（ノーポジ時）が存在していたが、
  クローズボタン（ロング中/ショート中時）が未実装だったため追加。
  クローズ時のpnl_pips計算も`pip_multiplier()`を使ってロング/ショート両対応で実装した。

- gspreadの`worksheet.spreadsheet`プロパティ（親Spreadsheetへの参照）を使うことで、
  `sheet`オブジェクトだけが渡された場合でも別シート（FXトレード履歴）への書き込みが可能になった。
  引数を増やさずに済んだ。

## 直し方

1. **`sync_fx.yml`**：`schedule`の2行をコメントアウト。自動実行を停止し、手動・ボタン操作のみに変更。

2. **`sync_fx.py`**（主要変更）：
   - インポートに`attach_rci`/`detect_rci_signal_series`/`RCI_PERIODS`/`requests`を追加。
   - `_INTERVAL_PERIODS = {"1h": "730d", "4h": "60d"}`を追加（yfinance取得期間の時間足別マッピング）。
   - `compute_pair_update()`に`interval`/`rci_periods`引数を追加。
     EMAクロス（`detect_cross_at()`）を`attach_rci()`＋`detect_rci_signal_series().iloc[-1]`に差し替え。
     `original_entry_price`をstep_position呼び出し前に保存してEXIT時のpnl計算に使用。
     戻り値に`event`/`rci_short_val`/`direction`/`original_entry_price`を追加。
   - `append_trade_history(spreadsheet, history_row)`を追加（「FXトレード履歴」シートに`append_row`）。
   - `send_trade_email(pair_name, action, direction, price, reason, pnl_pips)`を追加（Resend API）。
   - `update_fx_watchlist_with_signals()`に`interval`引数を追加。フェーズ2でイベント発生時に
     `append_trade_history()`と`send_trade_email()`を呼び出すよう変更。

3. **`app_fx.py`**：
   - `from datetime import datetime`と`append_trade_history, send_trade_email`を追加インポート。
   - `SIGNAL_INTERVAL_OPTIONS = {"4時間足（推奨）": "4h", "1時間足": "1h"}`を追加。
   - `st.tabs()`に「📒 トレード履歴」を追加（tab1〜tab3 → tab1〜tab4）。
   - 遠隔コントロールに`SIGNAL_INTERVAL_OPTIONS`セレクタを追加し、`update_fx_watchlist_with_signals()`に`interval=sig_interval`を渡す。
   - ポジション操作の「ロング中/ショート中」分岐に「🔴 ポジションをクローズ（手動）」ボタンを追加。
     クローズ時にSheets書き込み・`append_trade_history()`・`send_trade_email()`を呼び出す。
   - tab4に`load_trade_history()`（ttl=60のキャッシュ付き）を実装。
     決済回数・勝率・PF・純損益のサマリーと色付きDataFrameを表示。

4. **`app_fx.py` / `stock/app.py`**：チャート分析タブの下部指標マルチセレクトのデフォルトを
   `["RSI", "MACD"]`→`["RCI"]`に変更。

## 運用フロー（変更後）

```
① チャート分析タブで RCI を目視確認（デフォルト表示）
② 遠隔コントロールで時間足（4h推奨）を選択→「🔄 更新する」を押す
   → RCIシグナルが出ていれば自動でエントリー記録・Sheets書き込み・メール通知
③ ブローカーで実際に発注
④ 再度「🔄 更新する」を押すたびにエグジット条件を自動チェック
   （損切りpips/利確pips到達 or RCI+80から反転でクローズ記録・メール通知）
⑤ 手動でクローズする場合は「🔴 ポジションをクローズ（手動）」ボタン
⑥ 「📒 トレード履歴」タブで新規/2回目の確認・損益集計
```
