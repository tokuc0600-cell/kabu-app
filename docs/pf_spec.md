# プロフィットファクター分析 仕様書

最終更新: 2026-06-19

---

## 概要

EMAクロスシグナルを対象に、過去データからバックテストを実行し、
プロフィットファクター（PF）をはじめとする優位性指標を算出する。

---

## 計算指標の定義

### プロフィットファクター（PF）
```
PF = 総利益（勝ちトレードの利益合計）÷ 総損失（負けトレードの損失合計）
```
- PF > 1.5：優位性あり（実用レベル）
- PF > 2.0：優位性高い
- PF < 1.0：損益がマイナス（使えない戦略）

### 勝率（Win Rate）
```
勝率 = 勝ちトレード数 ÷ 全トレード数 × 100
```

### 平均利益・平均損失
```
平均利益 = 総利益 ÷ 勝ちトレード数
平均損失 = 総損失 ÷ 負けトレード数
リスクリワード比 = 平均利益 ÷ 平均損失
```

### 最大ドローダウン（MDD）
```
MDD = バックテスト期間中の資産曲線の最大下落幅（%）
```

## バックテストのロジック

### シグナル定義

**ゴールデンクロス（GC）= ロングエントリー**
- 条件：fastEMA が slowEMA を下から上に抜けた足
- エントリー：クロス確定の次の足の始値

**デッドクロス（DC）= ショートエントリー（またはロングイグジット）**
- 条件：fastEMA が slowEMA を上から下に抜けた足
- エントリー：クロス確定の次の足の始値

### イグジット定義（初期実装）
- 次のクロスが発生したらイグジット（ホールド型）
- 損切り・利食いは設定しない（シグナルの純粋な優位性を見る）

### 利益・損失の計算
```
FXの場合：pips単位で計算
  利益/損失 = (イグジット価格 - エントリー価格) × 方向 × 10000

株の場合：%単位で計算
  利益/損失 = (イグジット価格 - エントリー価格) ÷ エントリー価格 × 100
```

## 対象データ

### FX
- データソース：yfinance
- 対象通貨ペア：Google Sheets「FXウォッチリスト」から取得
- 対応時間足：1時間足（1h）、4時間足（4h）、日足（1d）
- 取得期間：直近2年（デフォルト）

### 日本株
- データソース：yfinance（銘柄コード.T形式）
- 対象銘柄：Google Sheets「ウォッチリスト」から取得
- 対応時間足：日足（1d）のみ
- 取得期間：直近2年（デフォルト）

## EMAパラメータ設定

| 設定名 | fastEMA | slowEMA | 主な用途 |
|--------|---------|---------|---------|
| デフォルト（FX） | 20 | 200 | 既存設定を継承 |
| 中期（FX） | 50 | 200 | トレンド確認 |
| 短期（株） | 5 | 25 | 既存設定を継承 |
| 中期（株） | 25 | 75 | スイング向け |

## 出力ファイル仕様

### CSVフォーマット（backtest/results/）
```
ファイル名: YYYYMMDD_[ticker]_[timeframe]_fast[n]_slow[n].csv

カラム:
  signal_date     : シグナル発生日時
  signal_type     : GC（ゴールデンクロス）/ DC（デッドクロス）
  entry_price     : エントリー価格
  exit_price      : イグジット価格
  profit_loss     : 損益（pips or %）
  result          : WIN / LOSS
  hold_bars       : 保有期間（足の本数）
```

### サマリーCSV
```
ファイル名: YYYYMMDD_summary.csv

カラム:
  ticker, timeframe, fast_ema, slow_ema,
  total_trades, win_rate, profit_factor,
  avg_profit, avg_loss, risk_reward, max_drawdown
```
