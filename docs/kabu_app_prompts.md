## Phase 1：チャート表示

```
CLAUDE.mdを読み込んで、プロジェクトの構成を確認してください。

app_fx.pyに以下のチャート表示機能を追加してください。

【追加内容】
- Plotlyを使ったローソク足チャートの表示
- 20EMA・200EMAのライン表示（既存の計算結果を流用）
- サイドバーまたはセレクトボックスで通貨ペアを選択できるUI
- チャートの時間足は現在と同じ4時間足

【注意事項】
- 既存のウォッチリスト表示機能は壊さないこと
- パッケージ追加が必要な場合は uv add を使うこと
- 後のPhaseでエントリー・エグジットのラインを重ねられるよう、
  チャート描画部分を関数として分離しておくこと

実装後、ローカルで動作確認できるコマンドも教えてください。
```

## Phase 2：トレード記録

```
CLAUDE.mdを読み込んで、プロジェクトの構成を確認してください。

以下のトレード記録機能を追加してください。

【Google Sheets の変更】
「kabu」ワークブックに「トレード記録」シートを新規作成し、
以下の列構成にしてください：
- 記録ID（自動採番）
- 通貨ペア
- 方向（Long / Short）
- エントリー日時
- エントリー価格
- エグジット日時
- エグジット価格
- 損益（pips）
- 区分（利確 / 損切 / 保有中）
- 原因分析メモ（自由記述）

【Streamlit の変更】
app_fx.pyに「トレード記録」タブまたはページを追加し、
以下のUIを実装してください：
- 新規記録の入力フォーム（上記項目をst.formで入力）
- 入力後にGoogle Sheetsへ自動書き込み
- 既存記録の一覧表示（Sheetsから読み込み）
- Phase1で実装したチャートに、選択したトレードの
  エントリー・エグジットの水平ラインを重ねて表示

【注意事項】
- 既存のウォッチリスト表示・チャート表示は壊さないこと
- 認証はst.secrets["gcp_service_account"]を流用すること
- パッケージ追加が必要な場合は uv add を使うこと
```

## Phase 3：アラート機能

```
CLAUDE.mdを読み込んで、プロジェクトの構成を確認してください。

sync_fx.pyにEMAクロス検出時のメール通知機能を追加してください。

【追加内容】
- sync_fx.py実行時に20EMA・200EMAのゴールデンクロス/デッドクロスを検出
- クロスが発生した通貨ペアをメールで通知
- 通知内容：通貨ペア名・クロスの種類・現在値・発生日時

【メール送信の実装方針】
Gmailを使ったSMTP送信で実装してください。
送信元アドレス・パスワードは .streamlit/secrets.toml から読み込む形にしてください。
（キー名の案：smtp_user, smtp_password, alert_to_email）

【注意事項】
- 前回実行時と比べてクロスが新規発生した場合のみ通知すること
  （毎回通知されないよう、最終シグナル状態をGoogle Sheetsで管理）
- パッケージ追加が必要な場合は uv add を使うこと
- secrets.tomlへの追記内容もサンプル付きで教えてください
```

## Phase 4：GIFアニメーション生成

```
CLAUDE.mdを読み込んで、プロジェクトの構成を確認してください。

app_fx.pyにトレードのGIFアニメーション生成・ダウンロード機能を追加してください。

【追加内容】
- Phase2で記録したトレードを選択すると、
  エントリー日時〜エグジット日時の4時間足ローソク足チャートを
  1本ずつコマ送りするGIFアニメーションを生成する
- 生成したGIFをStreamlitのダウンロードボタンで保存できるようにする

【GIFの仕様】
- 表示要素：ローソク足・20EMA・200EMAライン
  エントリー価格（緑の水平線）・エグジット価格（赤の水平線）
- フレームレート：1コマ = 4時間足1本（適度なアニメーション速度に調整）
- サイズ：横800px × 縦500px 程度
- ライブラリ：matplotlib + Pillow を使用すること

【注意事項】
- GIF生成は時間がかかるため、st.spinnerで生成中の表示を出すこと
- パッケージ追加が必要な場合は uv add を使うこと（pip install禁止）
- 生成したGIFはメモリ上で処理し、サーバーにファイル保存しないこと
```

## 保存場所のガイダンス

このファイルは以下のパスに保存することを推奨します：

```
kabu_app/
└── docs/
    └── kabu_app_prompts.md   ← このファイル
```

### 理由
- `docs/`フォルダはGitで管理しつつ、認証情報とは分離できる
- CLAUDE.mdと同じプロジェクトルートに近い場所に置くことで、
  Claude Codeがプロジェクトの意図を把握しやすくなる
- `.gitignore`の対象外なので、GitHubにも残せる

### 使い方
1. `kabu_app/docs/` フォルダを作成
2. このファイルをそこに保存
3. Claude Codeで作業開始時に「docs/kabu_app_prompts.mdのPhase1のプロンプトで実装してください」と伝える
